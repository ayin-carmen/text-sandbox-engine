# 阶段 10 世界实体编辑器使用说明

## 启动

先启动 Python API：

```powershell
cd E:\text-sandbox-engine
$env:PYTHONPATH = "src"
python -m text_sandbox_editor_api
```

另开终端启动前端：

```powershell
cd E:\text-sandbox-engine\editor
npm run dev -- --port 5173
```

打开 `http://localhost:5173`，输入内容包路径并点击“打开”。如果 Python API 或 Vite 进程已经运行过旧版本，修改代码后需要重启对应进程。

## 打开世界实体

1. 打开工作区后，点击顶部“世界实体”。
2. 左侧浏览器可以按全部、角色、地点、物品筛选。
3. 搜索框支持显示名称、稳定 ID 和标签。
4. 选择实体后，右侧显示结构化表单、revision 和引用影响。

实体数据继续保存于 `world_state.json.entities`，不会生成新的运行时文件。

## 创建角色

1. 点击右上角“新建实体”。
2. 实体类型选择“角色”。
3. 填写名称、命名空间、英文短标识和标签。
4. 选择初始地点。
5. 点击“预览并校验”，确认 ID 和模板结果。
6. 点击“确认创建”。

角色表单支持编辑显示名称、标签、当前地点和初始背包。角色 ID 创建后保持稳定，不能直接改写。

## 创建地点

1. 在向导中选择“地点”。
2. 填写名称、命名空间、英文短标识和标签。
3. 可选择一个初始连接地点。
4. 预览通过后确认创建。

地点表单支持名称、描述、区域、相邻地点和访问限制。相邻地点只能选择已有地点，避免写入无效连接。

## 创建物品

1. 在向导中选择“物品”。
2. 填写名称、命名空间、英文短标识和标签。
3. 预览通过后确认创建。

物品表单支持名称、描述、类别、是否可堆叠和最大堆叠数量。物品可以随后在角色背包和场景效果的引用选择器中使用。

## 编辑和保存

- 结构化表单适合修改常用字段。
- “JSON 高级模式”用于编辑暂未覆盖的组件字段。
- 修改后会执行防抖校验，错误会显示错误码和 JSON path。
- “保存”使用 revision 冲突检测、`.bak` 备份和原子替换。
- “撤销草稿”恢复最近一次读取或保存的版本。
- 未保存实体不会进入试玩会话。

## 引用影响和删除

实体编辑器会显示当前实体被哪些对象引用，包括：

- 角色的当前地点。
- 角色背包中的物品。
- 地点之间的连接。
- 场景所属地点。
- 场景条件和效果中的角色、地点、物品引用。

存在引用时，删除按钮会被禁用，后端也会拒绝删除。请先修复引用，或复制为新的稳定 ID。

## 与场景编辑联动

实体保存成功后，场景结构化表单的引用选择器会自动刷新。可以直接选择刚创建的角色、地点或物品，不需要刷新页面。

关系图会显示实体节点以及当前地点、背包物品和地点连接关系。试玩只读取已经保存的 `world_state.json`。

## 高级校验和测试

```powershell
cd E:\text-sandbox-engine
python -m unittest discover -s tests
python -m compileall -q src tests scripts

cd editor
npm test -- --configLoader runner
npm run build -- --configLoader runner
```

如果实体保存失败，先查看实体诊断中的错误码和 JSON path；不要直接覆盖 `world_state.json`，以免绕过引用保护和备份流程。
