# 阶段 9 低代码编辑器使用说明

## 启动

先启动本地 Python API：

```powershell
cd E:\text-sandbox-engine
$env:PYTHONPATH = "src"
python -m pip install -e ".[editor]"
python -m text_sandbox_editor_api
```

另开终端启动前端：

```powershell
cd E:\text-sandbox-engine\editor
npm install
npm run dev
```

打开 `http://localhost:5173`。如果 5173 已被已有 Vite 进程占用，直接使用已经运行的页面，或执行 `npm run dev -- --port 5174`。

## 创建场景

1. 在顶部输入内容包路径，例如 `E:\text-sandbox-engine\examples\medieval_town`，点击“打开”。
2. 点击左侧“新建场景”。
3. 在向导中填写场景名称和英文短标识。
4. 从地点选择器中选择所属地点。
5. 选择空白、普通叙事、NPC 对话、接取任务、交付物品或到达地点事件模板。
6. 设置重复策略和优先级，点击“预览并校验”。
7. 确认生成的场景 ID 后点击“确认创建”。如果 ID 已存在，向导会显示自动避让后的候选 ID。

## 配置场景逻辑

在“结构化表单”中：

- 条件、显示条件和效果类型由 Registry 提供中文名称和说明。
- 角色、地点、物品、任务、场景和 Flag 参数使用引用选择器。
- 整数、数字和布尔参数使用对应控件。
- 条件、显示条件、选项和效果支持复制、删除、上移和下移。
- 效果按照从上到下的顺序执行。

所有表单修改都会同步到同一份 JSON 草稿，但不需要手写 JSON。需要处理未被结构化控件覆盖的字段时，仍可切换到“JSON 源码”。

## 校验和保存

文档修改后会自动执行防抖校验。底部诊断信息包含错误码、JSON path 和修复建议；点击诊断会切回表单并定位到相关条件、显示条件或效果。

错误文档不能保存。保存前 API 会再次调用真实内容校验，并继续使用阶段 8 的 revision 冲突检测、临时文件原子替换和 `.bak` 备份。

切换场景、刷新页面或关闭窗口前，如果有未保存草稿，编辑器会提示确认。顶部“撤销草稿”可以恢复到最近一次读取或保存的版本。

## 按钮试玩

1. 切换到“运行预览”，点击“启动试玩”。
2. 编辑器显示当前角色、地点、时间、可前往地点和当前可见场景选项。
3. 点击地点或选项按钮执行真实 Python 引擎命令。
4. 查看中文业务摘要和状态变化。
5. 点击“重置会话”可以回到源 world state 的初始状态。

操作按钮来自本地 API 的真实引擎适配层，前端不重新实现规则。原始命令、Trace、场景候选、State Diff 和 Changed By 位于“高级详情”中。试玩只使用内存会话，不修改源 `world_state.json`。

## 高级模式

“JSON 源码”适合处理自定义字段或暂未被表单覆盖的结构。合法 JSON 会继续保留未知字段；JSON 语法错误时，编辑器保留源码并禁止保存，避免静默覆盖合法草稿。
