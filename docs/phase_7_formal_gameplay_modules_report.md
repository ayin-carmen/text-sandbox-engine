# 阶段 7：正式玩法模块扩展报告

阶段 7 根据阶段 6 垂直切片暴露出的真实需求，优先实现一组小而正式的玩法模块，而不是一次性铺开所有候选系统。

## 本阶段实现模块

1. `inventory`
   提供物品持有判断、添加物品、移除物品。

2. `social`
   提供信任值判断和信任值调整。

3. `quest`
   提供任务阶段判断、阶段设置和任务完成。

4. `narrative`
   补齐 `repeat_policy: once` 的已见场景记录与过滤。

5. `space`
   扩展地点准入规则，支持 `required_flag` 动态许可。

## 垂直切片验证

中世纪小镇切片已从“用旗标模拟玩法”升级为正式模块表达：

1. 面包师请求会给玩家添加 `item.bread_basket`。
2. 送货任务会进入 `quest.bread_delivery: accepted`。
3. 完成小教堂送货会移除面包篮并完成任务。
4. 艾尔达的信任值会随接受与完成送货增加。
5. 任务完成且信任达标后，会出现感谢场景并给玩家 `item.warm_bread`。
6. 领主塔楼通过 `required_flag: has_keep_pass` 表达动态准入。

## 模块化标准对照

1. 这些功能被多个内容文件重复使用：是。
2. 这些功能需要独立状态：是，包括物品、信任、任务阶段和已见场景。
3. 这些功能需要独立规则和效果：是。
4. 这些功能影响多个其他模块：是，任务影响场景，物品影响送货，社交影响感谢事件。
5. 这些功能需要独立测试与诊断报告：是，已新增阶段 7 模块测试。

## 暂缓模块

以下候选模块暂不实现：

1. `clothing`
2. `schedule`
3. `law`
4. `economy`
5. `content_filter`

原因：阶段 6 垂直切片没有暴露出这些模块的必要状态、规则和效果需求。后续应在内容制作中出现重复需求后再提升为正式模块。
